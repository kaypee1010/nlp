🚀 What is NLP, Really?

Natural Language Processing (NLP) is about making computers understand human language — the way we write and speak every day.

Think of it this way: When you ask Google "What is the capital of France?", the computer must understand your words, figure out what you're asking, and find the right answer. That whole process is NLP.

The Levels of Language Understanding

NLP works in layers — from the surface structure of words all the way to meaning:

LevelWhat it StudiesExample
MorphologyWord forms"running" → run + ing
SyntaxSentence structure (grammar)"The cat sat" — is this grammatically correct?
SemanticsMeaning of words & sentencesWhat does "bank" mean here?
PragmaticsMeaning in context / intent"Can you pass the salt?" = Please pass it.
DiscourseMulti-sentence understandingLinking "he" to "Rahul" in a paragraph

🧩 Syntax vs Semantics — The Core Difference

Syntax = Grammar rules. Is the sentence structured correctly?
Semantics = Meaning. Does the sentence make sense?

Classic Example

"Colorless green ideas sleep furiously."

This sentence is syntactically correct (follows grammar rules) but semantically meaningless (makes no real-world sense). This shows they are two different things!

SyntaxSemantics
FocusStructure / FormMeaning / Content
Question asked"Is it grammatical?""What does it mean?"
Tools usedCFG, Parse TreesWSD, SRL, Logic
Example correct"The cat sat on mat.""Bank" = riverside or financial?

📖 Key Terminology — Learn These First

Before we dive deep, let's build your vocabulary with these essential terms:

Parsing

Parsing = Breaking a sentence into its grammatical parts and showing how they relate to each other. Like labeling the skeleton of a sentence.

"The dog barked." → Subject (The dog) + Verb (barked)

Parse Tree

A Parse Tree is a diagram (tree shape) that shows how a sentence is broken down according to grammar rules — from the sentence level all the way down to individual words.

Ambiguity

Ambiguity = When a word or sentence has more than one possible meaning. This is one of the biggest challenges in NLP!

"I saw a bat." → A wooden bat? A flying animal?

Treebank

A Treebank is a large collection of sentences that have already been parsed by human experts. Computers use treebanks to learn how to parse new sentences.

📚 Think of a treebank as a textbook with solved examples that a computer studies to learn grammar.

📐 Context Free Grammar (CFG) Very Important

CFG is a set of rules that describe how sentences can be formed. It's the most common tool for parsing English sentences in NLP.

The 4 Components of CFG

ComponentSymbolMeaningExample
Non-terminalsNGrammar categories (can expand further)S, NP, VP, PP
TerminalsΣActual words (cannot expand)"dog", "runs", "the"
RulesRHow non-terminals expandNP → Det N
Start SymbolSThe top of every sentenceS (Sentence)

Common Non-Terminal Symbols

S → Sentence (the whole thing)
NP → Noun Phrase (e.g., "The big dog")
VP → Verb Phrase (e.g., "runs fast")
PP → Prepositional Phrase (e.g., "on the table")
Det → Determiner (a, an, the)
N → Noun (dog, girl, book)
V → Verb (eats, runs, reads)
Adj → Adjective (big, smart, red)
P → Preposition (on, in, with, by)

Writing CFG Rules — Step by Step

Rule format: LEFT SIDE → RIGHT SIDE
Read as: "A LEFT SIDE can be rewritten as RIGHT SIDE"

Let's build CFG rules for the sentence:

"The girl reads a book."
Step 1 — Start: a sentence has a NP + VP
S → NP VP

Step 2 — NP rules: a noun phrase can be Det + N
NP → Det N

Step 3 — VP rules: a verb phrase is V + NP
VP → V NP

Step 4 — Lexical rules: non-terminals → actual words
Det → "the" | "a"
N → "girl" | "book"
V → "reads"

More CFG Practice — "The boy eats an apple."

S → NP VP
NP → Det N
VP → V NP
Det → "the" | "an"
N → "boy" | "apple"
V → "eats"

📊 Statistical Parsing & PCFG

Normal CFG just tells us what's grammatically possible. But many sentences are ambiguous — there are multiple valid parse trees! Statistical parsing tells us which interpretation is most likely.

Probabilistic Context Free Grammar (PCFG)

A PCFG is a CFG where every rule has a probability attached to it. The probabilities for each non-terminal must add up to 1.0.

Example PCFG rules with probabilities:
NP → Det N [0.5] ← 50% of NPs are Det+N
NP → N [0.3] ← 30% of NPs are just a Noun
NP → Det Adj N [0.2] ← 20% of NPs are Det+Adj+N
——————
Total = 1.0 ✓
How it works: When two parse trees are both valid, the computer calculates the probability of each tree by multiplying all the rule probabilities used. The tree with the higher probability wins.

Lexicalized PCFG

A regular PCFG doesn't care about which specific words are used — only grammar categories. A Lexicalized PCFG also considers the actual words (the head word of each phrase) to make even better predictions.

For example: "ate with a fork" — "fork" is more likely to be an instrument than a subject, so the parser uses this word-level information to pick the correct structure.

🌳 Parse Trees — Reading & Drawing Very Important

A parse tree is like a family tree for a sentence. The root (top) is always S (Sentence), and we branch down to individual words at the leaves (bottom).

Rules for drawing parse trees:
① Always start at the top with S
② Branch into NP and VP
③ Keep expanding non-terminals (grammar categories)
④ The bottom row must be actual words (terminals)
⑤ Every word in the sentence must appear as a leaf

Example 1 — "The boy plays cricket."

Example 2 — "The girl reads a book."

How to Draw a Parse Tree — Step by Step

Step 1: Write S at the top.
Step 2: Split S into NP (subject) and VP (predicate).
Step 3: Expand NP → usually Det + N (or just N)
Step 4: Expand VP → usually V + NP (or just V)
Step 5: Continue expanding NPs inside VP the same way.
Step 6: At the bottom, replace each tag with the actual word.

🔀 Structural Ambiguity

Sometimes a sentence can be parsed in two different ways, giving two different meanings. This is called structural ambiguity.

"I saw the man with a telescope."

Meaning 1: I used a telescope to see the man.

Meaning 2: I saw the man who had a telescope.

In the first meaning, "with a telescope" is part of the VP (how I saw).
In the second meaning, "with a telescope" is part of the NP (which man).

This is why PCFGs and statistical methods are needed — to pick the more probable parse!

🧠 Semantic Analysis Very Important

Semantic analysis is how NLP systems extract and represent the meaning of language. It goes beyond grammar to understand what is actually being said.

What Does Semantic Analysis Do?

TaskExampleWhat it figures out
Word meaning"bank"River bank or financial bank?
Sentence meaning"The key opened the door."Key = instrument, door = theme
Relation finding"Paris is the capital of France."capital-of (Paris, France)
Inference"John broke the vase."The vase is now broken.

Applications of Semantic Analysis

🤖 Chatbots: Understanding what the user is asking (intent detection)
🎙️ Voice Assistants (Siri/Alexa): Extracting entities — "Set alarm for 7 AM"
🔎 Google Search: Finding pages that match the meaning of your query
🌍 Machine Translation: Preserving meaning across languages
❓ Question Answering: Finding the exact answer in a paragraph

📚 Lexical Semantics

Lexical semantics studies the meaning of individual words and the relationships between words.

Key Concepts in Lexical Semantics

ConceptDefinitionExample
SynonymyWords with same meaningbig = large = huge
AntonymyWords with opposite meaninghot ↔ cold
PolysemyOne word, multiple related meaningsbank (finance), bank (river)
HyponymyA word is a subtype of anotherrose IS-A flower
MeronymyPart-whole relationshipfinger IS-PART-OF hand

WordNet — The Lexical Database

WordNet is a famous dictionary-like resource used in NLP. It groups words into synsets (synonym sets) and records all these relationships. Many WSD systems use WordNet.

WordNet entry for "bank":
• Sense 1: a financial institution
• Sense 2: a long ridge of earth (river bank)
• Sense 3: to tilt sideways (aircraft banking)

🔗 Compositional Semantics

Compositional semantics is the idea that the meaning of a sentence is built up from the meanings of its individual words, following grammatical structure.

Principle of Compositionality:
"The meaning of a whole is a function of the meanings of its parts."

Example: "John loves Mary"

loves(John, Mary)
= loves is the predicate, John is the subject, Mary is the object

Example: "Every student passed the exam"

∀x. student(x) → passed(x, exam)
= For all x, if x is a student, then x passed the exam
Don't worry about the logic symbols for now — just remember the key idea: sentence meaning = sum of word meanings + structure.

💬 Discourse Analysis

Discourse analysis studies language beyond the single sentence level — how sentences connect to form coherent paragraphs and conversations.

What Problems Does Discourse Analysis Solve?

Coreference Resolution:
"Ravi went to the market. He bought vegetables."
→ Discourse analysis figures out that "He" = "Ravi"
Discourse Coherence:
Sentences in a text must logically connect. "It rained. I opened an umbrella." makes sense. "It rained. I ate a sandwich." is incoherent (no logical connection shown).

Real-Life Applications

ApplicationHow Discourse Analysis Helps
Automatic SummarizationIdentifies most important sentences across a document
ChatbotsRemembers context across multiple conversation turns
Machine TranslationEnsures pronouns and references are correctly translated

🔍 Ambiguity & Word Sense Disambiguation Very Important

What is Ambiguity?

Ambiguity occurs when a word, phrase, or sentence has more than one interpretation. In NLP this is a huge challenge because computers need a single, definite meaning.

Types of Ambiguity

TypeLevelExample
Lexical AmbiguityWord level"I saw a bat" — animal or cricket bat?
Structural AmbiguitySentence level"Flying planes can be dangerous."
Pragmatic AmbiguityContext level"Can you open the window?" — question or request?
"I went to the bank."
❓ Did the speaker go to a river bank for a picnic?
❓ Or did they go to a financial bank to deposit money?
Without more context, we can't tell. This is lexical ambiguity.

🎯 Word Sense Disambiguation (WSD) — 3 Approaches

WSD is the task of figuring out which sense/meaning of a word is intended in a given context.

① Supervised Approach

Train a machine learning model using labeled examples — sentences where humans have already tagged each word with the correct sense.

Training data examples:
"I deposit money in the bank." → SENSE: financial
"We picnicked on the river bank." → SENSE: geography
"The bank holds our savings." → SENSE: financial
The model learns: "deposit", "money", "savings" → financial sense
Supervised WSD
NeedsLarge labeled dataset
AccuracyHigh (when enough training data)
LimitationExpensive to label; may not handle new words well

② Dictionary-Based (Knowledge-Based) Approach

Uses a dictionary or lexical resource like WordNet to find the sense whose definition overlaps most with the surrounding words in the sentence.

Lesk Algorithm (classic example):
Context: "He flew to the bank to deposit his money."
→ Count overlap between context words and each WordNet definition.
→ "financial institution for depositing money" — high overlap ✓
→ "sloping land beside a river" — low overlap ✗
→ Chosen sense: financial
Dictionary-Based WSD
NeedsWordNet or similar dictionary
AccuracyModerate
AdvantageNo labeled training data needed

③ Unsupervised Approach

Groups word occurrences into clusters based on similar surrounding words — without any human labels. Each cluster represents a potential "sense".

Sentences with "bank" near "river", "water", "fish" → Cluster A (geography sense)
Sentences with "bank" near "loan", "account", "deposit" → Cluster B (financial sense)
Unsupervised WSD
NeedsLarge unlabeled corpus
AccuracyLower but useful when no labels exist
AdvantageCan discover new, unknown senses

SRL answers the question: Who did what to whom, where, when, and how? It assigns roles to the different parts of a sentence.

🗺️ Semantic Parsing

Semantic parsing is the process of converting a natural language sentence into a formal, machine-readable representation that a computer can execute or reason with.

Why Do We Need It?

"What movies were directed by Christopher Nolan after 2010?"
→ Human understands this easily.
→ But a database can only answer if we convert it to a query like:
SELECT title FROM movies WHERE director='Nolan' AND year > 2010

📊 Grammar Formalisms & Treebanks

Grammar Formalisms

A grammar formalism is a formal mathematical way of describing language structure. CFG is the most common, but others exist:

FormalismKey FeatureUse
CFG (Context-Free Grammar)Rules independent of contextStandard sentence parsing
PCFGCFG + probabilitiesResolving ambiguity statistically
Dependency GrammarWords linked by dependency arcsModern NLP parsers (spaCy)
CCG (Combinatory Categorical Grammar)Combines syntax + semanticsAdvanced semantic parsing

Treebanks

A treebank is a corpus of sentences annotated with parse trees. They are essential for training and evaluating parsers.

Penn Treebank — the most famous English treebank. Contains ~1 million words from Wall Street Journal articles, all parsed by human annotators.

Universal Dependencies — a multilingual treebank project covering 100+ languages.

Why Are Treebanks Important?

ReasonExplanation
Training dataParsers learn grammar patterns from annotated examples
EvaluationWe can measure parser accuracy against human annotations
Consistent annotationStandardized labels allow different systems to be compared

⚖️ Features and Unification

Sometimes CFG rules are too general — they allow grammatically wrong sentences like "The dogs runs." (plural subject with singular verb).

Feature Structures

We add features (attribute-value pairs) to grammar rules to capture constraints like number, person, gender:

NP[num=sing] → Det[num=sing] N[num=sing]
NP[num=plur] → Det[num=plur] N[num=plur]

Now "The dogs run" ✓ but "The dogs runs" ✗ (feature mismatch)

Unification

Unification is the process of combining two feature structures — they merge only if their features are compatible (no contradictions).

[num = sing] + [num = sing] = [num = sing] ✓ (same = compatible)
[num = sing] + [num = plur] = FAIL ✗ (contradiction = unification fails)
