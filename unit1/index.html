What is NLP?
Definition · Why it matters · Real-life examples
▾
Definition

Natural Language Processing (NLP) is a branch of Artificial Intelligence that enables computers to understand, interpret, and generate human language.

What does NLP combine?

Linguistics
Rules of language — grammar, meaning, structure
Probability
Maths to handle uncertainty in language
Computer Science
Algorithms that process text efficiently
Machine Learning
Learning patterns from data automatically

Simple Real-Life Examples

You see NLP every day!
  1. Google Translate — converts English to Hindi
  2. Gmail — suggests next words while typing
  3. Chatbots — answer customer service questions
  4. Siri / Alexa — understand your spoken words
  5. Spam filter — detects junk emails automatically
🎯 Exam Tip
If asked "what is NLP", always mention: AI branch + understand/interpret/generate language + combines linguistics + probability + CS + ML.

Quick Check ✓

Which of the following is NOT part of what NLP combines?

NLP combines Linguistics, Probability, Computer Science, and Machine Learning — not Chemistry!
Correct! Chemistry has nothing to do with NLP.

Levels of Language in NLP
Syntax · Semantics · Pragmatics
▾

Human language has three levels that NLP must handle. Think of them as layers — from structure to meaning to context.

LevelFocusNLP TaskExample
SyntaxSentence structureParsing, POS taggingGrammar rules
SemanticsWord meaningWord-sense disambiguationDictionary meaning
PragmaticsContext meaningSentiment analysisDetect sarcasm

Syntax — Structure of Sentences

Syntax

The grammatical structure of a sentence — how words must be arranged.

Example

✅ "The boy eats an apple." → Subject + Verb + Object — Correct syntax

❌ "Boy apple the eats." → Same words, but wrong order — no meaning!

💡 When Microsoft Word underlines a sentence in red — it is checking syntax.

Semantics — Meaning of Words

The Bank Problem

"He deposited money in the bank."

Which bank? 🏦 Financial bank? Or 🌊 River bank?

Semantic analysis resolves this using context. Google knows "Apple" = company when you search "iPhone".

Pragmatics — Meaning in Context

Reading Between Lines

Friend says: "It's very cold here."

Literal meaning: Temperature is low.

Pragmatic meaning: "Please close the window!"

💡 Chatbots use pragmatics to understand "I'm starving" = "Give me food suggestions".

🎯 Memory Trick
S-S-P → Structure (Syntax) → Sentence meaning (Semantics) → Purpose in context (Pragmatics). Think: "Structure, Sense, Purpose."

Quick Check ✓

"The computer understands that 'great' is sarcasm in a negative review." This is an example of which level?

Correct! Understanding sarcasm requires context → that's Pragmatics.
Sarcasm needs context to detect — that's the job of Pragmatics, not Syntax or Semantics alone.
03
Major Issues in NLP
5 reasons why language is hard for computers
▾

Natural language is messy. Here are the five main reasons why NLP is challenging:

1. Ambiguity

Classic Example

"I saw the man with a telescope."

Did I use a telescope to see the man? OR did the man have a telescope? Two completely different meanings — same sentence!

2. Synonymy

Same meaning, different words

Big = Large = Huge = Enormous

A search for "big house" should also find "large house" — computers need to know these mean the same thing.

3. Polysemy

One word, many meanings

Bat → Cricket bat OR flying animal

Bank → Financial bank OR river bank

Date → Calendar date OR a fruit OR a romantic outing

4. Language Variability

People speak differently!
Formal
"I do not know the answer."
Informal
"Dunno lol!"

Same meaning. NLP must handle both.

5. Sarcasm

Tone is invisible in text

"Oh great, the power is out again."

"Great" here is NOT positive. But a computer sees the word "great" and may classify this as positive sentiment!

Memory Aid — 5 Issues
Ambiguity · Synonymy · Polysemy · Variability · Sarcasm

Remember: "A Silly Parrot Very Slowly"

Role of Machine Learning in NLP
Why rules alone don't work
▾
Old Way: Rule-Based
Write grammar rules manually. Fails when language varies. Does not scale. Early chatbots used this.
New Way: ML-Based
Feed data, let computer find patterns. Flexible, scalable. Google Translate uses this.
Simple Analogy

Teaching a child to read:

Rule-based = Give the child a 500-page grammar book and say "follow these rules".

ML-based = Show the child 10,000 books. The child figures out the rules on their own!

ML Approach in Practice

  1. Collect labelled data (e.g., 10,000 spam + non-spam emails)
  2. Feed the data into an ML algorithm
  3. Algorithm detects patterns (certain words predict spam)
  4. Model can now classify new emails it has never seen
FeatureRule-BasedML-Based
UsesGrammar rulesData patterns
FlexibilityLowHigh
ScalabilityPoorGood
AdaptabilityHardEasy
🎯 Exam Tip
In exam, always mention the limitation of rule-based NLP: "Language is too complex and variable for fixed rules to cover all cases."

Probability Basics in NLP
The maths of uncertainty
▾

Language is uncertain. We don't know for sure what word comes next. So we use probability — a number between 0 and 1 showing how likely something is.

Basic Probability Formula
P(Event) = Favorable outcomes ───────────────── Total outcomes
📝 Worked Example: Word Frequency ▾ Show
Given
Word "the" appears 25 times. Total words = 500.
Apply
P("the") = 25 ÷ 500 = 0.05
Answer
The probability of "the" appearing is 5%.

Why Probability in NLP?

Instead of saying "the next word IS 'school'" (which we can't know for certain), we say:

Probabilistic Thinking

"After 'I am going to the', the probability of 'school' is 40%, 'market' is 30%, 'park' is 20%, 'moon' is 0.01%."

The model picks the word with the highest probability.

07
Information Theory & Entropy
Measuring uncertainty in language
▾

Information Content

How much "surprise" does a word carry? Rare words carry more information than common ones.

Intuition

If I say "I saw a the" — not surprising at all. "The" is very common → low information.

If I say "I saw a volcano" — very surprising! Rare word → HIGH information.

Information Content Formula
I(x) = − log₂ P(x) Low probability → High information High probability → Low information

Entropy — The Big Idea

Entropy H(X)

Entropy measures the average uncertainty in a system. In NLP: how unpredictable is the next word?

Key Rule — Very Important!

🔵 If something is very predictable → LOW entropy

🔴 If something is very unpredictable → HIGH entropy

Example: "The sun rises in the ___" → Almost certainly "east" → LOW entropy

Example: "I saw a ___" → Could be anything → HIGH entropy

Entropy Formula
H(X) = −Σ P(x) × log₂ P(x) i.e., sum over all possible outcomes of: (probability × log of that probability) then take the negative

Worked Examples — Important for Exam!

📝 Example 1: Fair Coin (H = 1 bit) ▾ Show
Given
P(Heads) = 0.5, P(Tails) = 0.5
Note
log₂(0.5) = −1
Apply
H = −[0.5 × (−1) + 0.5 × (−1)]
H = −[−0.5 − 0.5] = −[−1] = 1 bit
Meaning
Fair coin = maximum uncertainty = 1 bit entropy
📝 Example 2: Biased Coin (H < 1 bit) ▾ Show
Given
P(H) = 0.9, P(T) = 0.1
Values
log₂(0.9) ≈ −0.152    log₂(0.1) ≈ −3.32
Apply
H = −[0.9×(−0.152) + 0.1×(−3.32)]
= −[−0.1368 − 0.332] = −[−0.4688] ≈ 0.469 bits
Conclusion
Biased coin (more predictable) → LOWER entropy than fair coin ✓
📝 Example 3: Word Distribution (cat/dog/cow) ▾ Show
Given
P(cat)=0.5, P(dog)=0.3, P(cow)=0.2
Values
log₂(0.5)=−1   log₂(0.3)≈−1.737   log₂(0.2)≈−2.322
Apply
H = −[0.5(−1) + 0.3(−1.737) + 0.2(−2.322)]
= −[−0.5 − 0.5211 − 0.4644]
= −[−1.4855] ≈ 1.49 bits
⚠️ Key Property
Entropy is maximum when all outcomes are equally likely. It is zero when one outcome has probability 1 (certainty = no uncertainty).
🎯 Exam Tip
Always remember: Entropy = Uncertainty. Low entropy = model is confident. High entropy = model is confused. This connects directly to Perplexity (Topic 11).
08
Collocations
Words that love to be together
▾
Definition

Collocations are pairs (or groups) of words that frequently occur together as a fixed expression. You can't just swap one word for a synonym!

Examples

✅ Strong tea — we say this

❌ Powerful tea — sounds wrong, even though "strong" and "powerful" are synonyms

✅ Fast food — collocation

❌ Quick food — doesn't sound natural

✅ Make a decision — not "do a decision"

Why Collocations Matter in NLP

  • Auto-complete (Gmail, WhatsApp) predicts likely next words using collocations
  • Machine translation must respect natural pairings, not just translate word-by-word
  • Text generation sounds natural only when it respects collocation patterns
🎯 Exam Tip
Collocations are identified statistically — words that appear together far more often than chance would predict. This links to N-gram models.
09
N-gram Language Models
Predicting the next word with probability
▾
N-gram

A sequence of N consecutive words. We use the last N−1 words to predict what comes next.

TypeNExample from "I am fine"
Unigram1I / am / fine
Bigram2(I am) / (am fine)
Trigram3(I am fine)

How Bigram Prediction Works

Sentence: "I am learning NLP"

Bigrams extracted: (I → am) → (am → learning) → (learning → NLP)

So if someone types "I am going to ___", the model looks at "going to" and predicts most likely next word → e.g., "school".

Bigram Probability Formula
P(wn | wn-1) = Count(wn-1, wn) ──────────────── Count(wn-1) Read as: "Probability of word wn given the previous word wn-1"
📝 Worked Example: P(am | I) ▾ Show
Given
"I am" appears 20 times. "I" appears 25 times.
Apply
P(am | I) = Count("I am") ÷ Count("I") = 20 ÷ 25 = 0.8
Meaning
There is an 80% chance that "am" follows "I".

Advantages & Disadvantages

✅ Advantages
Simple to implement. Fast. Works well for short-range predictions.
❌ Disadvantages
Ignores long-distance context. Needs huge data. Memory intensive.
⚠️ Problem with N-grams
"The book that I bought yesterday was very ___" — N-gram only looks at nearby words. It misses that the subject "book" is far back in the sentence. This is the long-distance dependency problem.

Quick Check ✓

In a training corpus, "play cricket" appears 15 times and "play" appears 50 times. What is P(cricket | play)?

✓ Correct! P(cricket|play) = 15/50 = 0.3 → 30% chance.
Use the formula: Count("play cricket") ÷ Count("play") = 15 ÷ 50 = 0.3

Parameter Estimation & Smoothing
The zero probability problem & how to fix it
▾

The Zero Probability Problem

⚠️ Critical Problem
If a word combination never appeared in training data → its count = 0 → probability = 0 → the entire sentence probability becomes 0. This is called the Zero Probability Problem.
Example

Training data never contains the phrase "I love pizza".

So Count("love pizza") = 0 → P(pizza | love) = 0/count = 0

Then: P("I love pizza") = P(I) × P(love|I) × 0 = 0

The model says this sentence is impossible. That's wrong!

Solution: Add-One (Laplace) Smoothing

Laplace Smoothing

Add 1 to every word count — even words that never appeared. This ensures no probability is ever exactly zero.

Laplace Smoothing Formula
P(word | context) = Count(context, word) + 1 ───────────────────────────── Count(context) + Vocabulary Size
📝 Worked Example: Smoothing ▾ Show
Given
Count("I love") = 0, Total bigrams = 10, Vocabulary size = 5
Without Smoothing
P = 0/10 = 0 ← Problem!
With Smoothing
P = (0+1) / (10+5) = 1/15 ≈ 0.067 ← Small but NOT zero ✓
Benefit
The model can now handle new word combinations it never saw in training.
🎯 Exam Tip
Smoothing types you should know: (1) Add-One / Laplace — simplest; adds 1 to all counts. (2) Good-Turing — more advanced, redistributes probability. For BSc level, Laplace is sufficient.

Quick Check ✓

Why is zero probability a problem in N-gram models?

✓ Exactly! In N-gram models, probabilities are multiplied. Any zero term makes the whole product zero — making valid sentences appear impossible.
The key issue: sentence probability = product of word probabilities. Multiply any number by 0 and you get 0.
11
Evaluating Language Models
Perplexity — the gold standard metric
▾

How do we know if our language model is good? We measure it!

Metric 1: Perplexity

Perplexity

Perplexity measures how confused the model is when predicting words. Think of it as: "how many words is the model choosing between at each step?"

Classroom Analogy

Imagine a multiple-choice test:

• 2 options → Easy → low confusion → low perplexity ✅

• 100 options → Hard → high confusion → high perplexity ❌

Lower perplexity = better language model

Perplexity Formula
Perplexity = 2^H(X) where H(X) = entropy of the model Also: PP(W) = P(w1, w2, ..., wN)^(−1/N)
📝 Worked Example 1: Entropy → Perplexity ▾ Show
Given
Language model has entropy H(X) = 2 bits
Apply
Perplexity = 2^H(X) = 2^2 = 4
Meaning
The model behaves as if choosing between 4 possible words at each step.
📝 Worked Example 2: Sentence Perplexity ▾ Show
Given
Sentence "I love natural language processing" → N = 5 words
P(sentence) = 0.2×0.5×0.4×0.6×0.5 = 0.012
Apply
PP = P(W)^(−1/N) = (0.012)^(−1/5)
Answer
PP ≈ 2.41 → Model chooses between ~2.4 words per step. Good!
📝 Worked Example 3: P(W) = 0.008, N = 4 ▾ Show
Given
P(W) = 0.008, N = 4 words
Apply
PP = (0.008)^(−1/4) = (0.008)^(−0.25)
Answer
PP ≈ 3.34 → Model choosing between ~3.3 words per step.

Metric 2: Accuracy

In next-word prediction: how often does the model predict the correct word? Used for chatbots (response quality), autocomplete (accuracy), speech recognition (error rate).

The Golden Rule of Language Models
Lower Perplexity = Better Language Model

Perplexity = 2^H(X) · Entropy = −Σ P(x)log₂P(x)

Final Quiz — Unit 1

If Model A has perplexity 50 and Model B has perplexity 20, which is better?

✓ Correct! Lower perplexity = less confused model = better predictions. Model B wins!
Remember: Lower Perplexity = Better Model. Model B with 20 is less confused than Model A with 50.
