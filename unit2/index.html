01
Lexical Syntax
The building blocks of words
▾
Why do we start here?
Before processing sentences, a computer must understand individual words. Lexical syntax is the very first step.
Lexical Syntax

The study of words and their structure — what are the words, how are they formed, and what is their basic meaning.

Breaking Down a Word

Example: "Unhappiness"

Un → prefix meaning "not"

happy → root word (base meaning)

ness → suffix meaning "state of"

Combined: "state of not being happy"

More Examples

re + play + ing → replaying

un + break + able → unbreakable

pre + heat → preheat

⚠️ Limitation
Knowing individual word structures is not enough. We still don't know grammar or how words relate to each other in a sentence. That's where Morphology comes in.
🎯 Exam Tip
Lexical syntax answers: (1) What is the word? (2) How is it formed? (3) What is its basic meaning? — Know these three questions!
02
Morphology
How words change their form
▾
Why needed?
Words have multiple forms: play, playing, played, plays. A computer must know these are all the SAME word — just different forms.
Morphology

The study of the internal structure of words and how words are built from smaller units (morphemes).

Two Types of Morphology

TypeWhat it doesExample
Inflectional Changes tense, number, person — same base meaning play → played → playing → plays
Derivational Creates an entirely new word with new meaning happy → happiness → unhappy → unhappiness
Real-Life Analogy

Think of a word as a family. play, playing, played, player are all from the same family. Morphology is the family tree of words.

Why Morphology Matters in NLP

  1. Stemming — Reduces vocabulary size by mapping words to their root: "running" → "run"
  2. Lemmatization — More accurate root finding: "better" → "good"
  3. Better search — Search for "run" should find "running", "ran", "runner"
⚠️ Limitation
Morphology tells us about word structure but NOT about how words flow in a sequence. We need a model for that — leading to FST and then probabilistic models.
03
Finite State Transducers (FST)
A machine that converts one word form to another
▾
Problem it solves
How do we automatically convert: go → went, play → played, run → running? We need a systematic rule machine.
Finite State Transducer (FST)

A machine that takes an input word and produces an output word based on rules. It maps one linguistic form to another.

How FST Works

"play"
Input
→
FST
+ "ing" rule
→
"playing"
Output
More FST Examples

happy + [add 'ness'] → happiness

run + [double consonant + ing] → running

go + [irregular past] → went

Real-Life Analogy

Like a Vending Machine

Input: Insert coin (word)

Rule: Machine processes it

Output: Get snack (transformed word)

→ FST is a word-transformation machine!

Use in NLP

✅ Uses
Spell correction, morphological analysis, word form generation
❌ Limitation
Works only on fixed rules. Cannot handle uncertainty or ambiguity.
🎯 Exam Tip
FST = Input word → Rules → Output word. It's a two-level model: surface form (what we see) ↔ lexical form (dictionary form). Used in morphological analysis.
04
Part-of-Speech (POS) Tagging
Labelling every word's grammatical role
▾
The Problem
The word "play" can be a verb ("I play cricket") or a noun ("Let's watch a play"). The computer must decide which one — based on context.
POS Tagging

Assigning a grammatical label (Noun, Verb, Adjective, etc.) to every word in a sentence based on its role in that sentence.

POS Tags Reference

TagFull NameExample Word
NNNounboy, school, India
VBVerbruns, eats, plays
JJAdjectivebeautiful, tall, fast
RBAdverbquickly, silently
DTDeterminerthe, a, an
INPrepositionin, on, at, of
PRPPronounI, he, she, they
POS Tagging Example

Sentence: "The boy is playing cricket"

The → DT (Determiner)

boy → NN (Noun)

is → VBZ (Verb, 3rd person singular)

playing → VBG (Verb, present participle)

cricket → NN (Noun)

Why POS Tagging is Important

  1. Word sense disambiguation — "bank" as noun = financial, "bank" as verb = store
  2. Named entity recognition — proper nouns identify people, places
  3. Machine translation — word role changes translation
  4. Grammar checking — identifies incorrect sentence structure
🎯 Exam Tip
POS tagging approaches follow an evolution: Rule-Based → Markov → HMM → TBL → MaxEnt → CRF. Each solves a problem the previous couldn't. Know this sequence!
05
Rule-Based POS Tagging
The if-else approach
▾
Rule-Based POS Tagging

Assign POS tags using a large set of hand-written if-else grammar rules based on word endings, prefixes, and neighbouring words.

How Rules Work

Sample Rules

Rule 1: If word ends in -ing → Tag as Verb (VBG)

Rule 2: If word ends in -ness → Tag as Noun (NN)

Rule 3: If previous word = "the" → next word is likely a Noun

Rule 4: If word ends in -ly → Tag as Adverb (RB)

Rule 5: If word is in verb list → Tag as Verb

📝 Worked Example: Rule-based tagging ▾ Show
Input
"The quickly running dog"
Rule 1
"The" → previous = start → DT (Determiner)
Rule 2
"quickly" → ends in -ly → RB (Adverb)
Rule 3
"running" → ends in -ing → VBG (Verb) ← but context says it's an adjective here!
Problem
Rules don't understand context → "running dog" uses "running" as adjective, not verb. Rules fail here.
✅ Advantage
Simple to understand. Works well for common patterns. No training data needed.
❌ Limitations
Too many rules. Not scalable. Fails on ambiguity. Can't learn new language patterns.

Quick Check ✓

A rule says: "If word ends in -ed, tag it as past tense verb (VBD)." What would happen with the word "red" (the colour)?

✓ Correct! "red" ends in -ed but it's an adjective. Rule-based systems fail on exceptions like this — showing why smarter approaches are needed.
"red" ends in -ed so the rule would tag it as VBD (past verb) — which is wrong! This is exactly the limitation of rule-based systems.
06
Markov Models
Using probability instead of rules
▾
Key Insight
Instead of hard rules, use probability! The next word likely depends on the current/previous word. This is the Markov assumption.
Markov Model

A model where the probability of the next state depends only on the current state — not on the history of all previous states. Also called the "Markov assumption" or "memoryless property".

Markov Assumption
P(wn | w1, w2, ..., wn-1) ≈ P(wn | wn-1) "The next word depends only on the previous word, not the entire history"

Markov Chain Example

The
P=0.6
cat
P=0.4
sat
P=0.7
down
Real-Life Example

After the word "the", what likely comes next?

P(dog | the) = 0.15   P(cat | the) = 0.12   P(book | the) = 0.08

→ Markov model picks the most probable next word.

Contrast: P(run | the) = very low — "the" rarely precedes a verb directly.

✅ Advantage over Rules
Handles uncertainty. Flexible. Learns from data. More realistic than fixed rules.
❌ Limitation
Only considers visible/current state. Cannot model hidden factors (like POS tags that we don't directly see in raw text).
🎯 Exam Tip
The Markov assumption says: the future depends only on the present, not the past. This is also called "first-order Markov". A second-order Markov model looks at 2 previous words.
07
Hidden Markov Models (HMM)
Discovering what's hidden behind what we see
▾
The Key Problem
We can see the words in a sentence. But we CANNOT directly see the POS tags. The tags are "hidden". HMM solves this!
Hidden Markov Model (HMM)

A statistical model where we observe certain outputs (words) but the states that generate them (POS tags) are hidden. HMM finds the most likely hidden state sequence.

Two Key Probabilities in HMM

ProbabilityWhat it meansExample
Transition P P(next tag | current tag) P(Verb | Noun) = 0.7 (noun often followed by verb)
Emission P P(word | tag) P("runs" | Verb) = 0.05
The Ambiguous Sentence

"Time flies like an arrow"

This sentence has multiple valid interpretations! "flies" could be a verb or a noun. "like" could be a verb or preposition.

HMM calculates the probability of every possible tag sequence and picks the highest — the Viterbi algorithm does this efficiently.

Why HMM is Better than Plain Markov

Markov Model
Works with visible states only. POS tags must be directly observable.
HMM
Can infer hidden tags from observed words. Handles ambiguity probabilistically.
📝 HMM in Action — Step by Step ▾ Show
Task
Tag sentence: "Dog barks"
Candidates
Possible tag sequences: (NN, VBZ) or (NN, NN) or (VB, VBZ)...
Calculate
P(NN, VBZ | "Dog barks") = P(NN)×P("dog"|NN)×P(VBZ|NN)×P("barks"|VBZ)
Compare
Calculate probability for all sequences. Pick the highest.
Answer
Typically (NN, VBZ) wins → Dog/NN + barks/VBZ ✓
⚠️ HMM Limitation
Assumes independence between observations. Cannot use future words (right context) for tagging decisions. Limited context window. Leading to Transformation-based learning.

Quick Check ✓

In HMM for POS tagging, what are the "hidden" states and what are the "observed" outputs?

✓ Exactly! We observe the words but the POS tags are hidden — HMM infers them from the words using probability.
We can directly SEE the words (observed). We cannot directly see the POS tags — they are hidden. HMM discovers the hidden tags.
08
Transformation-Based Learning (TBL)
Start rough, then fix mistakes with rules
▾
Problem it solves
HMM makes tagging errors. What if we could automatically learn rules to fix those errors from training data? That's TBL!
Transformation-Based Learning (TBL / Brill Tagger)

A hybrid approach: start with an initial (rough) tagging, then automatically learn correction rules from training data to fix mistakes.

How TBL Works — 3 Steps

  1. Initial tagging — Assign the most frequent tag to each word (rough first pass)
  2. Compare with correct — Check against gold-standard (correct) tagged data
  3. Learn correction rules — Find transformations that fix the most errors, repeat
TBL Rule Example

Initial tag: "bank" → NN (noun) ← often wrong

Correction rule learned from data: "If the word is tagged NN AND the previous word is 'to', then change tag to VB (verb)"

Applied: "to bank" → "bank" becomes VB ✓ (to bank = to save money)

Another Rule

Rule: "If word is tagged VBD (past tense) but follows a determiner like 'the' or 'a', change to NN"

Applied: "the red" → "red" changed from VBD to JJ (adjective) ✓

✅ Advantage over HMM
Rules are human-readable. Combines statistical learning with linguistic rules. More accurate than pure HMM.
❌ Limitation
Still depends on hand-crafted rule templates. Cannot use all available features simultaneously.
09
Maximum Entropy Models
Using all features without assumptions
▾
Problem with Previous Models
HMM assumes words are independent of each other (Naive Bayes assumption). Real language uses many features simultaneously — we need a model that can use them ALL.
Maximum Entropy (MaxEnt) / Log-linear Model

Uses all available features simultaneously to predict the best POS tag — without assuming they are independent of each other.

Features Used in MaxEnt

Features for tagging the word "play"

1. The word itself: play

2. Previous word: they

3. Next word: cricket

4. Word ending: -ay

5. Is word capitalized: No

6. Previous POS tag: PRP (pronoun)

→ MaxEnt combines ALL these features to assign the best tag

The "Maximum Entropy" Principle

Core Idea

Among all probability distributions that satisfy our constraints (known features), choose the one with maximum entropy — the most uniform one that doesn't make unnecessary assumptions.

"Don't assume anything beyond what the data tells you."

✅ Advantage
Uses rich features. More accurate. No independence assumption. Flexible.
❌ Limitation
Computationally expensive to train. Does not model sequence dependencies well.
🎯 Exam Tip
MaxEnt = uses multiple features + no independence assumption + computationally expensive. The key advantage over HMM: it can use overlapping, dependent features without a problem.
10
Conditional Random Fields (CRF)
The state-of-the-art sequence labeller
▾
Why CRF is needed
MaxEnt tags each word independently. But POS tagging is a sequence problem — the tag of one word depends on tags of nearby words. CRF models the entire sentence together!
Conditional Random Field (CRF)

A model that assigns labels to a sequence of words by considering the entire sequence at once — not word-by-word. It combines the feature richness of MaxEnt with the sequence modeling of HMM.

CRF in Action

Why Sequence Matters

Sentence: "New York is beautiful"

If we tag word-by-word: "New" → JJ (adjective), "York" → NN (noun) → Two separate entities.

CRF looks at the whole sequence: "New York" together → single Named Entity (NNP NNP = proper noun) ✓

CRF vs Other Models

Context-aware. Rich features. Whole-sequence modeling. Best accuracy for NER, POS, chunking.

1. Rule-Based

↓ solved by...
6. Conditional Random Fields (CRF) ⭐
Rich features + whole sequence modeling + no false independence.
✅ State-of-the-art for sequence labeling

Final Unit 2 Quiz

Which model considers the ENTIRE sentence sequence when assigning POS tags?

✓ Correct! CRF is the only model that considers the entire sequence and uses rich features without independence assumptions — making it state-of-the-art.
HMM considers sequence but with limited features + independence assumption. MaxEnt uses rich features but NOT the sequence. Only CRF does BOTH.
